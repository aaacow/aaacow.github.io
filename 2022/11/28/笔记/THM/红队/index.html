查看网页源代码

页面源代码是每次我们发出请求时从 Web 服务器返回给我们的浏览器/客户端的可读代码。

返回的代码由 HTML（超文本标记语言）、CSS（层叠样式表）和 JavaScript 组成，它告诉我们的浏览器要显示什么内容，如何显示它并添加与 JavaScript 的交互元素。

如何查看

  1. 右击
  2. 在url前加入view-source:

要注意的几点

注释

这是HTML的注释语言

链接

这是定向链接语句

这两个一般会隐藏一些重要信息（重要提示，重要页面），在ctf比赛中很常见。

目录

可以使用 HTML 代码包含外部文件，例如 CSS、JavaScript 和图像。有可能这些文件都存储在同一目录中。如果您在 Web 浏览器中查看此目录，则会出现错误，显示的是一个空白页面或一个 403 禁止访问页面，说明您无权访问该目录。

但要是没有禁止访问的话，我们访问这个目录，就可以看到此目录下的所有文件，可能包含备份文件，重要代码或其他秘密信息。

框架

有些网站是使用框架搭建的，查看网页源代码可以让我们了解某个框架是否正在使用，如果正在使用，是哪个框架，甚至是哪个版本。框架中可能存在公共漏洞，并且网站可能没有使用最新版本。

开发者工具

inspector

页面源并不总是代表网页上显示的内容；这是因为 CSS、JavaScript 和用户交互可以更改页面的内容和样式，这意味着我们需要一种方法来查看此时浏览器窗口中显示的内容。Element inspector 通过向我们展示网站上当前的实时内容来协助我们。

就是对元素（elements）进行修改，调试，比如可以修改表单限制字数、去掉边框（block:none）等等。

debugger

在 Firefox 和 Safari 中，此功能称为 Debugger，但在 Google Chrome 中，它称为 Sources。

在谷歌浏览器左侧，可以看到当前网页正在使用的所有资源的列表。单机assets，将看到一个名为 flash.min.js 的文件。单击此文件会显示 JavaScript 文件的内容。

很多时候查看 javascript 文件时，您会注意到所有内容都在一行上，这是因为它已被最小化，这意味着所有格式（制表符、间距和换行符）都已被删除以使文件更小。这个文件也不例外，它也被混淆了，故意让它难以阅读，所以其他开发人员不能轻易复制它。可以点击下面的{}使其更具有可读性。

滚动到 flash.min.js 文件的底部，您将看到以下行：flash['remove']();

正是这段 JavaScript 代码从页面中删除了红色弹出窗口。我们可以利用调试器的另一个功能，称为 断点。它的作用是强制浏览器停止处理 JavaScript 并暂停当前执行的代码。点击代码前面的行号，会发现它变成了蓝色，表示已经插入了一个断点，再刷新一下，会注意到红色框保留在页面上而不是消失。
